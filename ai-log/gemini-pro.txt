階段一：進入 GRUB 開機選單

階段二：進入 Root 權限終端機
在 GRUB 選單中，你應該會看到 "Ubuntu"、"Advanced options for Ubuntu" 等選項。

你會看到一個新列表。請選擇結尾帶有 "(recovery mode)" 的那一行。

例如：Ubuntu, with Linux 5.15.0-xx-generic (recovery mode)

Bash

例如，如果你的使用者名稱是 brian，你就輸入：passwd brian

我推薦你安裝 tilix。它是一個非常受歡迎、功能強大、且外觀與 Gnome 系統很搭的終端機。

請在 xterm 裡面執行以下步驟：
安裝 Tilix： 在 xterm 裡面輸入 (或複製貼上) 以下指令，然後按 Enter：

Bash
sudo apt install tilix
它會問你密碼，你就輸入你的新密碼。

它會問你 [Y/n]，你就按 Y 然後按 Enter。

安裝完成後：

你可以關掉 xterm。

按下你的 Windows 鍵，在應用程式選單搜尋 "Tilix"。

點擊 Tilix 的圖示。

因為 tilix 是全新的安裝，它會用全新的設定檔，它有 99.9% 的機率會正常啟動。

你就可以把 tilix 釘選 (Pin) 到你的常用工具列，完全取代那個壞掉的 gnome-terminal，然後開心地繼續你的工作。

VM跟本機的剪貼簿可以共用嗎

是的，絕對可以！

要讓 VirtualBox 裡的 Ubuntu 和你本機的剪貼簿共用（雙向複製貼上），你需要完成兩個主要步驟：

在 VirtualBox 中開啟「共用剪貼簿」功能。

在你的 Ubuntu 虛擬機內部安裝「Guest Additions (客體額外功能)」。

步驟一：開啟「共用剪貼簿」設定
這是最簡單的一步。

在 VirtualBox 管理程式中，選中你的 Ubuntu 虛擬機 (請先不要開機，在關機狀態下設定)。

點擊「設定值 (Settings)」。

切換到「一般 (General)」分頁。

再點選「進階 (Advanced)」子分頁。

你會看到「共用剪貼簿 (Shared Clipboard)」的選項，請將它從「停用」改為「雙向 (Bidirectional)」。

(可選) 你也可以順便把隔壁的「拖放 (Drag'n'Drop)」也設為「雙向」，這樣你就可以直接拖檔案進出虛擬機了。

按下「確定」儲存。

步驟二：在 Ubuntu 內安裝「Guest Additions」
這是讓剪貼簿「動起來」的關鍵驅動程式。既然你之前已經修復了 sudo 和 Terminal (或 xterm)，這一步應該會很順利。

請啟動你的 Ubuntu 虛擬機，然後打開你的終端機 (Tilix 或 xterm)：

方法 A：最簡單的安裝方式 (推薦優先嘗試)
我建議你先試試這個方法，它使用 Ubuntu 內建的套件來安裝，最單純：

更新套件清單：

Bash
sudo apt update
安裝 Guest Additions 相關套件：

Bash
sudo apt install virtualbox-guest-x11
(它可能會一併安裝 virtualbox-guest-utils 和 virtualbox-guest-dkms，這是正常的)

重新啟動： 安裝完成後，必須重新啟動虛擬機：

Bash
sudo reboot
重啟後，等它進入桌面，試著在你本機複製一段文字，然後到 Ubuntu 虛擬機裡貼上看看。如果成功了，就大功告成了！

方法 B：如果方法 A 失敗 (完整的 CD 安裝法)
如果方法 A 失敗了，請執行這個「官方」安裝方法。

安裝編譯工具： Guest Additions 需要在你的 Ubuntu 裡「編譯」驅動程式，所以需要安裝工具 (你之前可能裝過了，但再執行一次也沒關係)：

Bash
sudo apt update
sudo apt install build-essential linux-headers-$(uname -r) dkms
插入 Guest Additions CD：

在 VirtualBox 虛擬機視窗的最上方選單中，找到「裝置 (Devices)」。

點擊「插入 Guest Additions CD 映像... (Insert Guest Additions CD image...)」。

執行安裝：

這個動作會在你的 Ubuntu 桌面跳出一個光碟圖示，並詢問你是否要「執行」。

如果它有跳出詢問，請點擊「執行 (Run)」，然後輸入你的密碼。

如果它沒有跳出詢問：

請打開你的「檔案」App。

在左側會看到一個光碟機，點進去。

在裡面找到一個叫做 autorun.sh 或 VBoxLinuxAdditions.run 的檔案。

在空白處按右鍵，選擇「在終端機中開啟」。

在終端機中輸入以下指令：

Bash
sudo ./VBoxLinuxAdditions.run
重新啟動：

等它跑完所有安裝程序後，同樣必須重啟虛擬機：

Bash
sudo reboot
完成以上步驟 (無論 A 或 B) 並重啟後，你的本機和 Ubuntu 虛擬機的剪貼簿就可以共用了！

你需要我針對哪個步驟再做詳細說明嗎？

ubuntu scp port22 沒辦法使用

你好，這是一個非常經典的問題！

scp 沒辦法在 Port 22 使用，99% 的情況下，這代表你的 Ubuntu 虛擬機上的 ssh 服務有問題。

scp (Secure Copy) 只是 ssh (Secure Shell) 服務之上的一個檔案傳輸功能。沒有 ssh，就沒有 scp。

你是指哪一種情況？
為了準確幫你除錯，我需要先知道你的「方向」：

情況 A： 你想從本機 (例如 Windows/Mac) scp 檔案 "到" 你的 Ubuntu 虛擬機。 (Ubuntu 是伺服器 Server)

情況 B： 你想從 Ubuntu 虛擬機 scp 檔案 "到" 你的本機或其他電腦。 (Ubuntu 是客戶端 Client)

我猜你非常有可能是**「情況 A」**，而新安裝的 Ubuntu 預設是不會安裝 SSH 伺服器的。

🚀 解決方案 (針對情況 A：Ubuntu 當作 Server)
請在你的 Ubuntu 虛擬機的終端機 (Tilix 或 xterm) 中執行以下檢查和修復步驟：

步驟一：安裝並啟用 openssh-server (最可能的病因)
安裝 SSH 伺服器：

Bash
sudo apt update
sudo apt install openssh-server
啟動並設定為開機自啟：

Bash
sudo systemctl start ssh
sudo systemctl enable ssh
檢查服務狀態：

Bash
systemctl status ssh
你應該會看到綠色的 active (running) 字樣。如果看到了，ssh 服務就跑起來了！

步驟二：檢查防火牆 (UFW)
如果 ssh 正在運行，但你仍然連不上，那很可能是被防火牆擋住了。

允許 SSH (Port 22) 通過防火牆：

Bash
sudo ufw allow ssh
(這條指令等同於 sudo ufw allow 22/tcp)

查看防火牆狀態：

Bash
sudo ufw status
如果你看到 Status: active，請確保 22/tcp (或 ssh) 出現在 ALLOW IN 的清單中。

如果狀態是 inactive，那防火牆就沒有開啟，不是問題所在。

步驟三：(VirtualBox 關鍵) 檢查你的網路設定
這是 VirtualBox 特有的問題！ 你的虛擬機如何連上網路，會直接決定你能不能 scp 進去。

在 VirtualBox 管理程式中，選中你的 Ubuntu 虛擬機，點擊「設定值」->「網路」。

查看你的「附加到 (Attached to)」是什麼模式？

如果是 NAT (預設模式)：

你的 Ubuntu 虛擬機沒有「真正」的 IP 位址，它躲在 VirtualBox 後面。

你無法從本機直接 scp 到它的 IP。

解法： 你必須設定「通訊埠轉發 (Port Forwarding)」。

在「網路」設定頁面，點開「進階」，點擊「通訊埠轉發」，新增一條規則：

名稱：ssh

主機 IP：127.0.0.1

主機連接埠：2222 (你可以自訂，例如 2222)

客體 IP：(留白)

客體連接埠：22

設定完後，你要 scp 的對象不是虛擬機的 IP，而是你的本機：

Bash
# -P 2222 (大寫 P) 是指定你剛剛設定的主機連接埠
scp -P 2222 /path/to/local/file [your_username]@127.0.0.1:/path/to/vm/destination
如果是 橋接介面卡 (Bridged Adapter)：

這是最簡單的模式。 你的虛擬機會從你的路由器那裡拿到一個獨立的 IP (例如 192.168.1.10)，和你的本機電腦平起平坐。

你只需要在 Ubuntu 終端機用 ip a 指令查到它的 IP。

然後就可以直接 scp：

Bash
# -p 22 (小寫 p) 是預設值，可以省略
scp /path/to/local/file [your_username]@[VM_IP_ADDRESS]:/path/to/vm/destination
如果是情況 B (Ubuntu 當作 Client)
這比較罕見，因為 openssh-client (包含 scp 指令) 幾乎都是預安裝的。

檢查 scp 指令是否存在：

Bash
which scp
如果沒有顯示路徑 (例如 /usr/bin/scp)，就表示沒安裝。

安裝 Client：

Bash
sudo apt install openssh-client
