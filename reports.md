# Swarm 作業實驗紀錄

## 實驗目的
本次作業主要是使用 Docker Swarm，並模擬兩個設備作為 **Manager** 和 **Worker** 的工作概念。

## 實驗環境與初期設定
- 原本想在實驗室分配的 container 上操作，但因為不確定 port 是否可以隨意使用，因此改為使用：
  - 桌上型電腦
  - Mac 筆電
- 初期問題：
  - 資料庫 (DB) 無法在 Mac 上正常使用。
  - Swarm 創建的 container 出現多種連線問題。
  - 從實驗室實體 IP 嘗試，改用手機網路，再改成家用 Wi-Fi，但兩邊仍無法互相 ping 通。
  
## 問題解決方案
- 改在家用電腦上部署兩個虛擬機 (VM) 以模擬多設備環境。
- 將 NAT 設定改為使用網路介面卡 (NIC) 作為 **bridge**，成功解決網路連線問題。

## Swarm 配置流程
1. 選定一個 VM 作為 **Manager**：
   - 執行 `swarm init` 初始化。
   - 大部分程式碼都部署在 Manager 上。
2. 另一個 VM 作為 **Worker**：
   - 執行加入 swarm 的指令即可。
   - 初期 Worker 不需承擔其他任務，但後來發現資料庫初始化有問題，需要 Worker 也建立 db folder 並放入 `init.sql` 做初始化。

## 後續問題
- 主要集中在測試階段，包括資料庫初始化與連線穩定性。
